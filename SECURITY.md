# Security policy

## Supported versions

| Version / branch | Supported |
| ------------------ | --------- |
| `main` (latest)    | Yes       |

## Reporting a vulnerability

Please do **not** open a public issue for security problems. Open a private security advisory in this repository (see GitHub’s **Security** tab), or contact the maintainers with details and a preferred disclosure timeline.

## Expectations

- We will confirm receipt, assess severity, and coordinate a fix and release.
- Public disclosure is deferred until a patch is available unless otherwise agreed.

For architectural notes on deployment and secrets, see [docs/SECURITY.md](docs/SECURITY.md).
